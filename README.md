# ServiceDesk

ServiceDesk - веб-приложение для регистрации и обработки пользовательских заявок.

Финальная версия beta v3 сохраняет функции beta v2, добавляет REST API и использует две роли: пользователь и администратор.

## Функции

- регистрация пользователя или администратора;
- вход и выход из аккаунта;
- личный кабинет;
- создание заявки через веб-форму;
- список заявок пользователя;
- подробная страница заявки;
- админ-панель `/admin`;
- роль выбирается при регистрации;
- обычный пользователь видит только свои заявки;
- обычный пользователь создает заявки;
- администратор видит все заявки;
- администратор обрабатывает заявки и не создает их;
- администратор может менять статус заявки;
- администратор может написать или изменить ответ на заявку;
- пользователь видит ответ администратора на странице своей заявки;
- удаление заявок;
- загрузка файлов к заявке;
- разрешенные расширения файлов: `png`, `jpg`, `jpeg`, `pdf`, `txt`;
- страницы ошибок `403` и `404`;
- REST API для списка, просмотра, создания и удаления заявок.

## Запуск

Установите зависимости:

```bash
pip install -r requirements.txt
```

Запустите приложение:

```bash
python app.py
```

Откройте в браузере:

```text
http://127.0.0.1:8080
```

База данных создается автоматически в папке `data`.
Загруженные файлы сохраняются в папке `static/uploads`.

## API

API работает с текущей авторизованной сессией пользователя и возвращает JSON.

### GET /api/tickets

Возвращает список заявок.

- обычный пользователь получает только свои заявки;
- администратор получает все заявки.

Пример ответа:

```json
[
  {
    "id": 1,
    "title": "Не работает принтер",
    "text": "Нужна проверка",
    "category": "Принтер",
    "priority": "Обычный",
    "status": "Новая",
    "admin_answer": "",
    "file_name": null,
    "created_date": "2026-05-13T10:00:00",
    "user_id": 1,
    "user": {
      "id": 1,
      "name": "Admin",
      "email": "admin@example.com"
    }
  }
]
```

### GET /api/tickets/&lt;id&gt;

Возвращает одну заявку.

- обычный пользователь может получить только свою заявку;
- администратор может получить любую заявку.

### POST /api/tickets

Создает заявку через JSON.

Создавать заявки может только обычный пользователь. Для администратора API вернет `403`.

Обязательные поля:

- `title`
- `text`
- `category`
- `priority`

Пример запроса:

```bash
curl -X POST http://127.0.0.1:8080/api/tickets \
  -H "Content-Type: application/json" \
  -d "{\"title\":\"Тестовая заявка\",\"text\":\"Проверка API\",\"category\":\"Другое\",\"priority\":\"Обычный\"}"
```

Пример успешного ответа: статус `201`.

### DELETE /api/tickets/&lt;id&gt;

Удаляет заявку.

- обычный пользователь может удалить только свою заявку;
- администратор может удалить любую заявку.

Пример ответа:

```json
{
  "message": "Ticket deleted",
  "id": 1
}
```

### Ошибки API

Пример ошибки:

```json
{
  "error": "Access denied"
}
```

Основные статусы:

- `400` - неверный JSON или не заполнены обязательные поля;
- `401` - пользователь не вошел в аккаунт;
- `403` - нет доступа;
- `404` - заявка или страница не найдена.

## Тестовый сценарий

1. Запустите проект командой `python app.py`.
2. Откройте `http://127.0.0.1:8080`.
3. Откройте регистрацию и выберите роль `Пользователь`.
4. Зарегистрируйте обычного пользователя и войдите под ним.
5. Проверьте навигацию: `Кабинет`, `Мои заявки`, `Создать заявку`, `Выход`.
6. Создайте заявку через веб-форму.
7. Откройте список заявок и подробную страницу заявки.
8. Выйдите из аккаунта.
9. Откройте регистрацию и выберите роль `Администратор`.
10. Зарегистрируйте администратора и войдите под ним.
11. Проверьте навигацию: `Админ-панель`, `Выход`.
12. Откройте личный кабинет администратора через `/profile` и проверьте текст про админ-панель.
13. Откройте `/admin` и проверьте, что видны все заявки.
14. Измените статус заявки в админ-панели.
15. Напишите ответ администратора и сохраните заявку.
16. Войдите под обычным пользователем и откройте свою заявку.
17. Проверьте, что на странице заявки виден ответ администратора.
18. Создайте новую заявку и проверьте текст `Администратор пока не ответил на заявку.`
19. Удалите тестовую заявку в админ-панели.
20. Проверьте, что администратор не может открыть `/tickets/new`.
21. Проверьте страницу `404`, открыв несуществующий адрес.
22. Проверьте страницу `403`, открыв чужую заявку под обычным пользователем.
23. Проверьте API через браузер или `curl` с cookie авторизованной сессии.

## Проверка ответа администратора

1. Зарегистрируйте обычного пользователя.
2. Войдите под обычным пользователем и создайте заявку.
3. Откройте заявку и проверьте текст `Администратор пока не ответил на заявку.`
4. Выйдите и войдите под администратором.
5. Откройте `/admin` или подробную страницу заявки.
6. Выберите статус и заполните поле `Ответ администратора`.
7. Сохраните изменения.
8. Войдите под обычным пользователем и откройте заявку.
9. Проверьте, что ответ администратора отображается на странице заявки.

## Проверка API через curl

Для запросов к API нужна авторизованная сессия. Удобный вариант:

1. Войти в аккаунт через браузер.
2. В инструментах разработчика скопировать cookie `session`.
3. Передать cookie в запросе:

```bash
curl http://127.0.0.1:8080/api/tickets \
  -H "Cookie: session=PASTE_SESSION_VALUE"
```

Создание заявки:

```bash
curl -X POST http://127.0.0.1:8080/api/tickets \
  -H "Content-Type: application/json" \
  -H "Cookie: session=PASTE_SESSION_VALUE" \
  -d "{\"title\":\"API заявка\",\"text\":\"Создано через API\",\"category\":\"Другое\",\"priority\":\"Обычный\"}"
```

Получение одной заявки:

```bash
curl http://127.0.0.1:8080/api/tickets/1 \
  -H "Cookie: session=PASTE_SESSION_VALUE"
```

Удаление заявки:

```bash
curl -X DELETE http://127.0.0.1:8080/api/tickets/1 \
  -H "Cookie: session=PASTE_SESSION_VALUE"
```
